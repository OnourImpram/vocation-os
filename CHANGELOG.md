## 0.3.0 - 2026-07-10

Decision intelligence release. The theory list becomes an operational registry of 28 cited lenses with constructs, runtime decision questions, mode bindings, and rubric bindings, validated by `validateTheoryRegistry` and generated into `docs/THEORY_MAP.md`. Adds the career decision difficulty intake classifier, the option value reversibility rationale, and the `/skill-coach` psychoeducation mode pairing planned happenstance skills with acceptance and commitment processes.

Adds a structurally R0 advisory layer with public verified claim filtering, mode applicable theory filtering, private claim withholding, prompt injection red team coverage, explicit data classification, remote egress approval, HTTPS and host allowlist enforcement, redirect rejection, timeout, JSON content validation, and response size limits.

Adds pure Greenhouse, Lever, Ashby, and manual opportunity adapters with source hashes and deterministic fingerprints. Adds remote status, applicant geography, freshness, apply route, HTTPS, description quality, and duplicate intake gates.

Adds hash bound submission proof for confirmation pages, ATS dashboards, Sent Items, and receipt emails. Application attempts now remain `submitted_unconfirmed` until sufficient official proof is bound to the same opportunity. Security codes, incomplete notices, unverified send calls, missing attachments, tampered proof, and mismatched opportunities cannot create confirmed ledger entries.

Adds four operational schemas, two CLI demos, four evaluator cases, an independent implementation plan, a concrete threat model, an offline CycloneDX SBOM check, and a GitHub release evidence workflow using provenance and SBOM attestations.
